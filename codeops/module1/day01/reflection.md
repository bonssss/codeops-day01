# Reflection

## 1. What is the difference between Git and GitHub?

* **Git** is a version control system used to track changes in files and manage project history on your computer.
* **GitHub** is a cloud-based platform that hosts Git repositories, making it easy to store code online, collaborate with others, and manage projects.

## 2. What is the purpose of the terminal?

The terminal is a command-line interface that allows users to interact with the operating system by typing commands. It is commonly used to navigate directories, manage files, run programs, and execute Git commands.

## 3. What does `git add` do?

The `git add` command stages new or modified files, preparing them to be included in the next commit.

## 4. What is the purpose of `git commit`?

The `git commit` command saves the staged changes as a snapshot in the Git repository along with a descriptive commit message.

## 5. Write the daily Git workflow in the correct order.

1. Check the repository status (`git status`).
2. Edit or create files.
3. Stage the changes (`git add`).
4. Commit the changes (`git commit -m "message"`).
5. View the commit history if needed (`git log --oneline`).
6. Push the commits to GitHub (`git push`).
